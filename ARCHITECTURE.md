# 企业级问数系统 - 架构设计文档

## 1. 系统概述

企业级问数系统是一个基于 DeepAgents 的智能数据查询平台，通过自然语言处理技术，让用户能够用日常语言查询企业数据，无需编写 SQL。

### 核心价值
- 🎯 **零 SQL 门槛** - 业务人员可直接查询数据
- 🛡️ **企业级安全** - 多层安全防护和审计
- ⚡ **高性能** - 缓存、优化、连接池等多重优化
- 🤖 **AI 驱动** - 基于 DeepAgents 的智能 SQL 生成
- 📊 **多数据源** - 支持 MySQL、PostgreSQL、ClickHouse 等

## 3. 核心模块设计

### 3.1 Agent 引擎 (agent.py)

```
┌─────────────────────────────────────────────────────────┐
│                   DataQueryAgent                        │
├─────────────────────────────────────────────────────────┤
│  配置：AgentConfig                                      │
│  - model: GPT-4 / Claude / DeepSeek                    │
│  - role: SQL_EXPERT / DATA_ANALYST / ...               │
│  - temperature, max_tokens, timeout                    │
├─────────────────────────────────────────────────────────┤
│  核心方法：                                             │
│  - initialize()          # 初始化                       │
│  - set_schema_context()  # 设置 Schema                  │
│  - generate_sql()        # 生成 SQL                     │
│  - validate_sql()        # 验证 SQL                     │
├─────────────────────────────────────────────────────────┤
│  指标：AgentMetrics                                     │
│  - total_requests, success_rate                        │
│  - avg_response_time, cache_hit_rate                   │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│                     AgentPool                           │
├─────────────────────────────────────────────────────────┤
│  - 负载均衡：基于响应时间选择最优代理                   │
│  - 故障转移：代理故障时自动切换                         │
│  - 角色路由：根据查询类型选择对应角色代理               │
│  - Schema 广播：统一更新所有代理的 Schema 上下文            │
└─────────────────────────────────────────────────────────┘
```

### 3.3 SQL 处理组件 (sql_generator.py)

```
┌─────────────────────────────────────────────────────────┐
│                    SQLGenerator                         │
├─────────────────────────────────────────────────────────┤
│  SQL 模板库：                                           │
│  - simple_select    - aggregation    - time_series     │
│  - ranking          - comparison     - join            │
│  - trend            - filter                           │
│                                                         │
│  生成策略：                                             │
│  1. 优先使用 AI 生成 (如果有配置)                          │
│  2. 回退到规则生成 (模板 + 参数填充)                      │
│  3. 后处理和优化                                        │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    SQLValidator                         │
├─────────────────────────────────────────────────────────┤
│  安全检查：                                             │
│  ┌──────────────┬────────────────────────────┐         │
│  │ 风险等级     │ 检测操作                   │         │
│  ├──────────────┼────────────────────────────┤         │
│  │ Critical     │ DROP, TRUNCATE, DELETE     │         │
│  │ High         │ INSERT, UPDATE, ALTER      │         │
│  │ Medium       │ LOCK, SET                  │         │
│  │ Low          │ SELECT, WITH, EXPLAIN      │         │
│  └──────────────┴────────────────────────────┘         │
│                                                         │
│  注入检测：                                             │
│  - SQL 注释模式 (--, /* */)                              │
│  - OR 注入 (' OR '1'='1)                                │
│  - UNION 注入                                          │
│  - 存储过程执行 (EXEC, xp_cmdshell)                    │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    SQLOptimizer                         │
├─────────────────────────────────────────────────────────┤
│  优化策略：                                             │
│  1. SELECT * → 明确列名                                │
│  2. 添加/优化 LIMIT                                    │
│  3. 相关子查询 → JOIN                                  │
│  4. OR 条件 → UNION/IN                                  │
│  5. DISTINCT → GROUP BY                                │
│                                                         │
│  索引建议：                                             │
│  - WHERE 条件列                                         │
│  - JOIN 条件列                                          │
│  - ORDER BY 列                                          │
│  - GROUP BY 列                                          │
└─────────────────────────────────────────────────────────┘
```
